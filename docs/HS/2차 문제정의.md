문제 정의 및 관측 데이터

문제 현상:

interview_graph 실행은 실패하지 않았지만, 최종 세션 상태가 COMPLETED가 아니라 PARTIAL_COMPLETED로 저장되는 케이스가 반복적으로 관측되었다. 대시보드에서는 해당 세션들이 “일부 완료”로 표시되며, HR 담당자 입장에서는 질문 생성이 끝났는지, 추가 검토가 필요한지 구분이 필요하다.

관측 데이터:

항목	관측 내용	비고
실행 결과	partial_completed	LangGraph 최종 응답 상태
저장 상태	PARTIAL_COMPLETED	interview_sessions.question_generation_status
질문 생성 여부	질문은 저장됨	완전 실패는 아님
주요 의심 조건	승인 질문 수 부족	approved_count < 5
추가 조건	fallback 경고, 질문 수 부족, 문서 텍스트 없음, 재시도 한계 도달	final_formatter_node 기준

원인 분석

직접 원인:

nodes.py (line 1051)의 final_formatter_node에서 아래 조건 중 하나라도 참이면 partial_completed가 된다.

python



is_partial = (
    not has_extracted_text
    or len(items) < 5
    or approved_count < 5
    or state.get("retry_count", 0) >= state.get("max_retry_count", 3)
    or bool(state.get("node_warnings"))
)



즉 현재 현석 그래프는 “질문이 생성되었는가”가 아니라, 최종 질문 5개가 모두 승인 가능한 품질인가를 기준으로 완료 여부를 판단한다.

핵심 원인 후보:

번호	원인 후보	설명
H1	최종 승인 질문 수 부족	reviewer가 5개 중 일부를 needs_revision 또는 rejected로 판정하면 approved_count < 5
H2	재시도 한계	questioner 재시도는 기본 1회라, 한 번 보정 후에도 승인 5개를 못 채우면 일부 완료
H3	fallback 발생	predictor, driller, reviewer 실패 시 fallback을 사용하고 node_warnings가 생겨 일부 완료 처리
H4	최종 질문 수 부족	selector 이후 최종 질문이 5개 미만이면 일부 완료
H5	문서 추출 텍스트 없음	지원자 문서의 extracted_text가 없으면 질문 품질이 불완전하다고 판단

reviewer가 최종 질문 5개를 모두 approved로 판정하지 못하면, router가 재시도를 시도한다. 하지만 max_questioner_retry_count가 기본 1회이므로, 한 번의 재생성 후에도 품질 조건을 만족하지 못하면 selector로 종료되고 final_formatter에서 partial_completed가 된다.
