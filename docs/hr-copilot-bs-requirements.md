# 📄 HR Copilot BS — 기능 요구사항 정의서 (FR 기준)

> 기준: 테이블 정의서 기반
> 작성 목적: 개발/기획/운영 공통 기준 정립

---

# 1. FR코드 구조 및 모듈 정의

| 코드 | 모듈명 | 의미 |
|:---:|:---|:---|
| **FR-01** | 관리자 계정/권한 | 시스템 접근 제어 및 관리자 정보 관리 |
| **FR-02** | 메뉴 및 권한 매핑 | 계층형 메뉴 구성 및 그룹별 접근 권한 설정 |
| **FR-03** | 문서 관리 | 지원자 서류(이력서 등) 업로드 및 텍스트 추출 |
| **FR-04** | 프롬프트 관리 | LLM 전략 수립을 위한 템플릿 및 버전 관리 |
| **FR-05** | LLM 실행 | 실제 AI 분석 프로세스 및 워크플로우 제어 |
| **FR-06** | 면접 결과 관리 | 분석된 데이터 기반 질문 세트 생성 및 결과 저장 |
| **FR-07** | 로그/통계 | 운영 효율을 위한 비용, 성능, 호출 이력 추적 |
| **FR-08** | RAG 처리 | 문서 청킹 및 임베딩을 통한 근거 기반 데이터 추출 |
| **FR-09** | 공통 시스템 | 시스템 전반에 적용되는 공통 정책(Audit, Delete 등) |

# 2. 기능 요구사항

# FR-01 관리자 계정 / 권한 관리

| 코드 | 기능명 | 구분 | 상세 설명 | 우선순위 |
|:---|:---|:---:|:---|:---:|
| **FR-01-01** | 관리자 계정 등록 | 기능 | 관리자 계정 생성 (비밀번호 BCrypt 암호화 저장) | 상 |
| **FR-01-02** | 관리자 로그인 | 기능 | ID/PW 인증 및 계정 상태(Status) 검증 | 상 |
| **FR-01-03** | 관리자 상태 관리 | 기능 | ACTIVE / INACTIVE / LOCK 상태 변경 제어 | 중 |
| **FR-01-04** | 권한 그룹 생성 | 기능 | 권한 그룹 등록 (group_key UNIQUE 제약 적용) | 중 |
| **FR-01-05** | 권한 그룹 관리 | 기능 | 권한 그룹 수정 및 Soft Delete 기반 삭제 | 하 |

## 3. 📌 시스템 핵심 요약

1.  **데이터 파이프라인:** 문서 업로드(FR-03) → 청킹/임베딩(FR-08) → 텍스트 추출
2.  **AI 오케스트레이션:** 프롬프트 설계(FR-04) → 분석 실행(FR-05) → 운영 모니터링(FR-07)
3.  **서비스 인터페이스:** 메뉴/권한(FR-01, 02) → 면접 결과 제공(FR-06)
